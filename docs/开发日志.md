## 2026-06-20 - 内存模式下采集任务不执行问题修复

### 问题描述
点击"开始采集"后，进度一直是0，任务不执行。

### 问题分析
系统有两种任务执行模式：
1. **Redis 模式**：任务通过 RQ 队列入队，由独立的 worker 进程执行
2. **内存模式**：任务只是创建了并保存在内存字典中，但没有执行逻辑

当前环境 Redis 未启用，使用内存模式，任务创建后没有后台线程去执行，所以进度一直是 0。

另外，Streamlit 每次页面刷新都会重新运行整个脚本，全局变量 `task_manager` 会被重新创建，导致任务状态丢失。

### 修复方案
1. **添加内存模式后台工作线程**：在 `RedisTaskManager` 中添加 `_run_memory_worker` 方法，用后台线程循环检查并执行待处理的任务
2. **添加线程安全保护**：使用 `threading.Lock` 保护任务字典的并发访问
3. **使用 `@st.cache_resource`**：确保任务管理器是单例的，避免 Streamlit 刷新页面时重复创建实例，保证后台线程和任务状态不丢失

### 修改文件
- `tasks.py` - 添加内存模式后台工作线程、线程安全保护
- `app.py` - 使用 `@st.cache_resource` 确保任务管理器单例

## 2026-06-20 - 任务状态显示不同步问题修复

### 问题描述
后台日志显示任务已完成（新增 14 条），但前端还是显示"状态: running，已完成: 0/0"。

### 问题分析
1. **状态名称不统一**：内存模式任务完成后设置的状态是 `'finished'`，Redis 模式 RQ 返回的状态也是 `'finished'`，但前端检查的是 `'completed'`，导致状态不匹配，进度和完成数量都显示不正确。
2. **读取任务状态未加锁**：多线程环境下，读取任务状态时也需要加锁，确保数据一致性。

### 修复方案
1. **统一状态名称**：在 `get_task_status` 方法中，把 RQ 的 `'finished'` 统一转换成 `'completed'`，与前端保持一致。
2. **读取加锁**：`get_task` 方法读取内存任务时也加锁，确保线程安全。
3. **统一逻辑**：`get_task_status` 不再区分 Redis 和内存模式，使用统一的处理逻辑。

## 2026-06-20 - 编辑面板空 div 问题修复

### 问题描述
用户点击论文卡片上的"✏️ 编辑"按钮后，编辑面板显示为空，只有一个空的灰色框，里面没有任何内容。

### 问题分析
这是 Streamlit 开发中的一个常见陷阱：

用 `st.markdown('<div class="edit-panel">', unsafe_allow_html=True)` 手动写了一个 div 开始标签，
但后面的 `st.selectbox`、`st.button` 等 Streamlit 组件**不会被包含在这个 div 里面**。

Streamlit 的每个组件都是独立渲染的，有自己的容器，它们和平级的 markdown 内容是兄弟关系，而不是父子关系。

实际渲染结果：
```html
<div class="edit-panel"></div>  <!-- 空的！ -->
<div class="stSelectbox">...</div>  <!-- 组件在外面 -->
<div class="stButton">...</div>
```

所以用户看到一个空的面板，实际内容都在面板下面。

### 修复方案
使用 CSS `:has()` 选择器 + 隐藏标记元素的方式：

1. **移除手动的 div 标签**：删除 `<div class="edit-panel">` 和 `</div>`
2. **添加隐藏标记元素**：在编辑面板开头添加一个隐藏的 span：
   ```python
   st.markdown('<span class="edit-panel-marker" style="display:none;"></span>', unsafe_allow_html=True)
   ```
3. **用 :has() 选择器应用样式**：
   ```css
   [data-testid="stVerticalBlock"]:has(.edit-panel-marker) {
       background: #f8f9fa;
       padding: 1rem;
       border-radius: 8px;
       margin: 1rem 0;
       border-left: 4px solid #667eea;
   }
   ```

这样样式会正确应用到包含标记元素的 Streamlit 容器上，所有编辑面板的内容都在面板内部。

### 修改文件
- `app.py` - 修复编辑面板渲染问题，改用 :has() 选择器实现面板样式

实现"记住我"功能，用户登录后刷新页面或关闭浏览器再打开，仍保持登录状态。换设备登录需要重新输入账号密码。

### 实现方案
基于 localStorage + JWT Token 的自动登录机制：

1. **登录时存储 Token**：
   - 用户勾选"记住登录状态"后，登录成功时将 JWT token 写入浏览器 localStorage
   - Token 有过期时间（默认 24 小时），过期后自动失效

2. **自动登录流程**：
   - 页面加载时检测到用户未登录
   - 用 JavaScript 读取 localStorage 中的 auth_token
   - 如果有 token，将其加到 URL 参数中并重定向
   - 后端从 URL 参数读取 token，验证有效性
   - 验证通过则自动设置登录状态，清除 URL 参数中的 token

3. **退出登录**：
   - 清除 localStorage 中的 token
   - 数据库端标记会话失效
   - 清除 session_state 中的登录状态

4. **换设备失效**：
   - Token 存储在浏览器 localStorage 中，不同设备/浏览器不共享
   - 换设备后 localStorage 中没有 token，需要重新登录

### 修改内容
1. **登录页面**：
   - 添加"记住登录状态"复选框（默认勾选）
   - 登录成功后根据勾选状态决定是否写入 localStorage

2. **退出登录**：
   - 添加清除 localStorage 中 token 的逻辑

3. **自动登录**：
   - 页面加载时检查 URL 参数中的 token
   - 验证 token 有效则自动登录
   - 用 JS 读取 localStorage 中的 token 并触发自动登录

## 2026-06-21 - 记住登录状态功能最终实现

### 问题描述
之前的自动登录方案（URL参数方式）因 Streamlit 组件 iframe 的安全限制无法工作，具体表现为：
1. `st.components.v1.html` 在 iframe 中执行，无法直接修改父窗口 URL（报 SecurityError）
2. 改用隐藏输入框方案后，JS 设置 input 值无法触发 Streamlit 状态更新
3. CSS `:has()` 选择器使用不当导致整个页面白屏

### 调试过程
1. **第一版（URL参数重定向）**：失败 —— iframe 无 `allow-top-navigation` 权限，无法修改 `window.parent.location.href`
2. **第二版（隐藏输入框 + nativeSetter）**：部分成功 —— token 能填入输入框，但需用户手动点击触发才能生效
3. **第三版（CSS 优化）**：失败 —— `div:has(input[...])` 选择器匹配所有祖先 div，导致整个页面被隐藏白屏
4. **第四版（移出屏幕 + execCommand）**：成功 —— 用 `document.execCommand('insertText')` 模拟真实用户输入，配合 `position: fixed` 移出屏幕隐藏输入框

### 最终实现方案

**核心技术栈**：localStorage + JWT Token + 隐藏输入框 + execCommand 模拟输入

1. **Token 存储**：
   - 登录成功后通过 `window.parent.localStorage.setItem()` 将 JWT token 存入主页面 localStorage
   - 使用 `st.components.v1.html` 的 iframe 访问 `window.parent` 操作主页面存储

2. **自动登录流程**：
   - 页面加载时若未登录，渲染一个隐藏的 text_input（placeholder 为 `__auto_login_token__`）
   - CSS 用 `position: fixed; left: -9999px` 将输入框移出屏幕（保持可交互但不可见）
   - JS 从 localStorage 读取 token，用 `document.execCommand('insertText')` 模拟用户输入
   - Streamlit 检测到输入框值变化，后端验证 token 有效性
   - 验证通过则自动登录，验证失败则清除 localStorage 中的无效 token

3. **关键技术点**：
   - 不能用 `display: none` 隐藏输入框（元素无法获得焦点，execCommand 失效）
   - 必须用 `execCommand` 模拟真实输入（直接设置 value 无法触发 React 状态更新）
   - CSS 选择器需精确到 `[data-testid="stTextInput"]` 避免误隐藏外层容器

4. **退出登录**：
   - 清除 localStorage 中的 token
   - 数据库端使会话失效
   - 清除 session_state 登录状态

#### 1. 隐藏输入框（移出屏幕方式）
**核心原理**：不能用 `display: none`（元素无法获得焦点，JS输入失效），用移出屏幕的方式保持可交互性。

    top: -9999px !important;
    opacity: 0 !important;
    pointer-events: auto !important;
    z-index: -1 !important;
}
</style>
""", unsafe_allow_html=True)

# 隐藏输入框
st.text_input(

#### 2. JS 模拟用户输入（触发 Streamlit 状态更新）
**核心原理**：直接设置 `input.value` 无法触发 React 状态更新，必须用 `execCommand` 模拟真实输入。

```javascript
function setStreamlitInput(input, value) {
    if (input.value === value) return;
    
    input.focus();
    input.select();
    
    try {
        // 优先用 execCommand 模拟真实输入（最可靠）
        document.execCommand('delete');
        document.execCommand('insertText', false, value);
    } catch(e) {
        // 降级方案：nativeSetter 方式
        const nativeSetter = Object.getOwnPropertyDescriptor(
            HTMLInputElement.prototype, 'value'
        ).set;
        nativeSetter.call(input, value);
        input.dispatchEvent(new Event('input', { bubbles: true }));
    }
    
    input.dispatchEvent(new Event('change', { bubbles: true }));
}
```

#### 3. Streamlit 组件中访问主页面 localStorage
**核心原理**：`st.components.v1.html` 在 iframe 中执行，需通过 `window.parent` 访问主页面。

```javascript
// 读取
const token = window.parent.localStorage.getItem('auth_token');

// 写入
window.parent.localStorage.setItem('auth_token', token);

// 删除
window.parent.localStorage.removeItem('auth_token');
```

#### 4. 完整自动登录流程
```python
def main():
    current_user = st.session_state.get('current_user')
    
    if not current_user:
        # 检查自动登录 token
        auto_token = st.session_state.get('auto_login_token_input', '')
        
        if auto_token:
            # 验证 token
            payload = auth_manager.verify_token(auto_token)
            if payload:
                user = auth_manager.get_user_by_id(payload['user_id'])
                if user:
                    st.session_state.auth_token = auto_token
                    st.session_state.current_user = {
                        'id': user.id,
                        'username': user.username,
                        'full_name': user.full_name,
                        'email': user.email
                    }
                    st.rerun()
                    return
            else:
                # token 无效，清除 localStorage
                st.session_state.auto_login_token_input = ''
                st.components.v1.html("""
                <script>
                try { window.parent.localStorage.removeItem('auth_token'); } catch(e) {}
                </script>
                """, height=0)
        
        # 渲染登录页面
        render_login()
        
        # 隐藏输入框 + 自动填充 JS
        st.markdown("""
        <style>
        [data-testid="stTextInput"]:has(input[placeholder="__auto_login_token__"]) {
            position: fixed !important;
            left: -9999px !important;
            top: -9999px !important;
            opacity: 0 !important;
            pointer-events: auto !important;
            z-index: -1 !important;
        }
        </style>
        """, unsafe_allow_html=True)
        
        st.text_input(
            "auto_token", 
            key="auto_login_token_input", 
            label_visibility="collapsed",
            placeholder="__auto_login_token__"
        )
        
        st.components.v1.html("""
        <script>
        setTimeout(function() {
            try {
                const token = window.parent.localStorage.getItem('auth_token');
                if (token) {
                    const input = window.parent.document.querySelector(
                        'input[placeholder="__auto_login_token__"]'
                    );
                    if (input && input.value !== token) {
                        input.focus();
                        input.select();
                        window.parent.document.execCommand('delete');
                        window.parent.document.execCommand('insertText', false, token);
                        input.dispatchEvent(new window.parent.Event('change', { bubbles: true }));
                    }
                }
            } catch(e) {
                console.error('Auto login error:', e);
            }
        }, 1000);
        </script>
        """, height=0)
        return
```

## 2026-07-03 - 数据源采集功能实现

根据《数据库结构与数据源说明》文档中明确指定的14个数据源，设计并实现了完整的数据采集功能。
未纳入文档中标记为"未定"的16个数据源。

### 实现内容

#### 1. 配置文件更新（config.py + config/settings.yaml）
- 新增 `crawler` 配置节（max_workers、timeout、retry_times、retry_backoff）
- 新增 `sources` 配置节（15个数据源的启用状态和参数）
- 新增 `rss_feeds` 配置节（7个RSS订阅源地址）

#### 2. 数据采集模块（crawler.py）
基于备份代码重构，实现10个采集器类覆盖14个已确认数据源：

**API数据源（3个）**：
- `ArxivCrawler` - arXiv预印本（arxiv Python库）
- `OpenAlexCrawler` - 开放学术图谱（REST API）
- `PubmedCrawler` - PubMed生物医学（NCBI E-utilities）

**RSS数据源（4个）**：
- `ScienceDailyCrawler` - Science Daily科技新闻
- `MITTRCrawler` - MIT Technology Review（RSSCrawler子类）
- `TechCrunchCrawler` - TechCrunch科技媒体（RSSCrawler子类）
- `RSSCrawler` - 通用RSS/Atom订阅源（RSS-*）

**网页爬虫数据源（7个）**：
- `GenericWebCrawler` - 通用网页采集器，覆盖6个中文网站 + Nature
  - 中国科普博览（kepu.net.cn）
  - 中国数字科技馆（cdstm.cn）
  - 国家科技传播中心（cccst.org.cn）
  - 中关村论坛（zgcforum.com）
  - 科技导报（kjdb.org）
  - 中国科学技术馆（cstm.cdstm.cn）
  - Nature（nature.com）
- `ChineseJournalsCrawler` - 中文学术期刊采集器

**采集管理器（CrawlerManager）**：
- ThreadPoolExecutor并发调度（默认4线程）
- 支持实时进度回调（progress_callback）
- 支持停止检测（stop_check_callback）
- 异常隔离（单个采集器失败不影响其他）

**数据格式化**：
- `format_paper_data()` - 统一格式化（日期标准化、文本清洗、URL补齐、内容哈希）

**关键词翻译**：
- `translate_keywords()` - 中文→英文自动翻译（40+映射项）
- 英文API数据源自动使用翻译后的关键词查询

#### 3. 任务队列模块（tasks.py）
- `RedisTaskManager` - 双模式任务管理器（Redis + RQ / 内存队列）
- `create_crawl_task()` - 创建采集任务
- `get_task_status()` - 任务状态查询
- 内存模式下自动启动后台工作线程执行任务
- 采集任务自动入库（add_papers + add_crawl_history）
